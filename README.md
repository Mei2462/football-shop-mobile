Tugas 8: Flutter Navigation, Layouts, Forms, and Input Elements
 
1. Jelaskan perbedaan antara Navigator.push() dan Navigator.pushReplacement() pada Flutter. Dalam kasus apa sebaiknya masing-masing digunakan pada aplikasi Football Shop kamu?
-> Navigator.push menempatkan halaman baru di atas tumpukan. Sehingga dengan menekan tombol kembali, user dapat kembali ke halaman sebelumnya. Ini sangat cocok untuk fitur browsing atau eksplorasi. Misalnya pada aplikasi Football Shop saat user mengetuk kartu Create Product pada ItemCard di shop_card.dart dari beranda, itu sudah memakai push, sehingga user dapat kembali ke beranda setelah melihat atau membatalkan formulir saat user sekadar mencoba lalu mungkin batal.

Navigator.pushReplacement mengganti halaman saat ini dengan halaman baru, sehingga user sudah tidak bisa kembali ke halaman lama. Hal ini dilakukan karena kita tidak ingin menumpuk home berkali-kali di back stack, dan saya sudah menerapkan pola ini di LeftDrawer untuk Home. Pola ini juga ideal untuk proses yang melibatkan menyelesaikan satu langkah alur seperti setelah user sukses membuat produk pada Create Product maka arahkan ke My Products memakai pushReplacement agar tidak kembali ke form dan mencegah submisi ulang.

2. Bagaimana kamu memanfaatkan hierarchy widget seperti Scaffold, AppBar, dan Drawer untuk membangun struktur halaman yang konsisten di seluruh aplikasi?
-> Dengan menggunakan Scaffold sebagai widget yang berbeda yang memiliki AppBar dan LeftDrawer yang sama di setiap layar utama dan kemudian digunakan ulang di berbagai halaman aplikasi, saya dapat membuat pengalaman navigasi terasa seragam di seluruh aplikasi. 
a. AppBar menampilkan judul sementara warna dan tipografi diambil dari Theme.of(context) agar selaras dengan brand dan tidak hardcoded. Pada MyHomePage AppBar tampil dengan warna yang diambil dari Theme melalui Theme.of(context).colorScheme.primary sehingga terhubung dengan pengaturan tema dari MaterialApp. Sedangkan ProductsFormPage AppBar tetap hadir namun warnanya saya tetapkan langsung sehingga hierarki tetap sama meski sumber warnanya berbeda.
b. LeftDrawer sebagai pengganti tab bar untuk aplikasi katalog sederhana sehingga struktur halaman terasa menyatu di seluruh aplikasi. Navigator.pushReplacement digunakan sehingga back stack tetap bersih dan snackbar dari ScaffoldMessenger selalu muncul pada halaman yang sedang aktif.
c. Khusus beranda MyHomePage menggunakan Wrap atau Flexible untuk tiga InfoCard saat lebar layar sempit agar tidak terjadi overflow dan biarkan GridView.count menampilkan menu cepat di bagian tengah untuk konsistensi visual.

3. Dalam konteks desain antarmuka, apa kelebihan menggunakan layout widget seperti Padding, SingleChildScrollView, dan ListView saat menampilkan elemen-elemen form? Berikan contoh penggunaannya dari aplikasi kamu.
-> Dalam kode saya, setiap field dibungkus Padding di dalam Kolom sehingga memiliki tampilan yang rapi:
a. Padding membuat elemen input tidak berdempet dengan tepi layar dan membuat fokus pengguna lebih jelas. 
b. SingleChildScrollView mencegah overflow saat konten tinggi atau ketika keyboard muncul, dan telah diterapkan di products_form.dart sehingga form tetap dapat digulir pada perangkat/tampilan kecil. Pada halaman Create Product, form dibungkus SingleChildScrollView dengan Padding agar aman dari keyboard lalu field ditata rapi menggunakan Column. 

4. Bagaimana kamu menyesuaikan warna tema agar aplikasi Football Shop memiliki identitas visual yang konsisten dengan brand toko?
-> Untuk tema dan warna, saya telah menetapkan Theme di root MaterialApp dengan ColorScheme, yang berasal dari swatch biru muda, dan mengaktifkan Material 3. Penggunaan warna berbasis tema ini tampak di AppBar pada MyHomePage yang mengambil primary dari colorScheme. Namun, masih ada juga beberapa bagian lain yang masih memakai warna tetap (hardcoded) yaitu pada AppBar dan tombol simpan pada halaman form yang menggunakan indigo serta DrawerHeader yang menggunakan biru sehingga identitas visual saat ini merupakan kombinasi antara warna dari Theme dan warna yang ditetapkan langsung.
